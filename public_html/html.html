a

Es una etiqueta utilizada para vínculos a otras páginas web.

<a>...</a>

Atributos de a

href

Indica la dirección de la página web.

<a href="https://www.google.com"> Google </a>

Esto es un enlace: Google.

target

Las atributos target especifican dónde abrir el documento vinculado.

El atributo target puede tener uno de los siguientes valores:

  • _blank - Abre el documento vinculado en una nueva ventana o pestaña.

  • _self - Abre el documento vinculado en la misma ventana/pestaña en la que se ha hecho clic (predeterminado para los links).

  • _parent - Abre el documento vinculado en el marco padre.

  • _top - Abre el documento vinculado en el cuerpo completo de la ventana.

  • framename - Abre el documento vinculado en un marco con nombre.

Haz click en este link sin soltarlo

Para cambiar estos estilos y otro más, cambiaremos los apartados link, visited, hover y active.

  • link: como se muestra el link sin haberse visitado y sin tocarlo.
  • visited: como se muestra el link una vez visitado.
  • active: como se muestra el link cuando se está haciendo click sobre él.
  • hover: como se muestra el link cuando está el ratón encima de él pero sin hacer click.

Rutas de Ficheros

Las rutas de archivos son usados cuando se enlazan a un archivos externos como:

  • Páginas Web
  • Imágenes
  • Hojas de Estilo
  • JavaScripts

Hay 2 tipos de rutas posibles a ficheros.

Absoluta

Es una ruta utilizada de manera que se escribe la ruta del fichero completa desde la unidad.

src="C:\Windows\Users\usuario\html\img\icono.png"

Relativa

Es una ruta utilizada de manera que se escribe la ruta del fichero partiendo de la ruta en la que se encuentra el fichero desde el cuál se está haciendo referencia al fichero destino.

src="../img/icono.png"

Layouts

Una página web se divide en diferentes zonas.

  • header - Define una cabezera para un documento o una seccion.

  • nav - Define un contenedor para enlaces de navegación.

  • section - Define una seccion en un documento.

  • article - Define un artículo autónomo independiente.

  • aside - Define el contenido aparte del contenido (como una barra lateral)

  • footer - Define un pie de página para un documento o una sección.

Símbolos

Se pueden mostrar símbolos presentes y no presentes en un telado normal, tales como ∑

Cada símbolo tiene un número y un nombre el cúal los identifica.

La sintaxis para el número es &#numeros;

La sintaxis para el nombre es &nombre;

URL Enconde

Una URL identifica un documento u otro tipo de datos en una web

Su sintaxis es scheme://prefix.domain:port/path/filename

  • scheme - Define el tipo de servicio de Internet (los más comunes son http y https)

  • prefix - Define el prefijo del dominio (por defecto para http es www)

  • domain - Defines el nombre de dominio de Internet (como w3schools.com)

  • port - Define el número del puerto en el cliente (por defecto para http es 80)

  • path - Define una ruta en el servidor (si se omite: la ruta raíz del sitio)

  • filename - Define el nombre de un documento o recurso.

Formularios

Sirven para recoger datos introducidos por el usuario.

Elementos de los formularios

input

Es el elemento más importante de los formularios.

type

Indica de que forma podrá el usuario introducir la información.

text

Mediante una caja de texto

<form>
      <label for="fname">First name:</label><br>
      <input type="text" id="fname" name="fname"><br>
      <label for="lname">Last name:</label><br>
      <input type="text" id="lname" name="lname">
</form>



password

Permite ocultar los caracteres escritos sustituyéndolos por asteriscos a la vista del usuario.

<form>
      <label for="contraseñaEjemplo">Contraseña</label>
      <input type="password" name="contraseñaEjemplo" id="contraseñaEjemplo">
</form>
radio

Mediante botones tipo radio.

<form>
      <input type="radio" id="male" name="gender" value="male">
      <label for="male">Male</label><br>
      <input type="radio" id="female" name="gender" value="female">
      <label for="female">Female</label><br>
      <input type="radio" id="other" name="gender" value="other">
      <label for="other">Other</label>
</form>


checkbox

Mediante botones tipo checkbox.

<form>
      <input type="checkbox" id="coche" name="mediotransporte" value="coche">
      <label for="coche">Coche</label><br>
      <input type="checkbox" id="bicicleta" name="mediotransporte" value="bicicleta">
      <label for="bicicleta">Bicicleta</label><br>
      <input type="checkbox" id="autobus" name="mediotransporte" value="autobus">
      <label for="autobus">Autobús</label>
</form>


submit

Envía la información de los campos a rellenar o botones a seleccionar a un form-handler, que es una página en el servidor con un script que procesa la información.

La ruta al archivo form-handler se indica con el atributo action en el elemento form.

<form action="/action_page.php">
      <label for="fname">First name:</label><br>
      <input type="text" id="fname" name="fname" value="John"><br>
      <label for="lname">Last name:</label><br>
      <input type="text" id="lname" name="lname" value="Doe"><br><br>
      <input type="submit" value="Submit">
</form>





button

Botón el cuál le das una acción que hacer.

<form>
      <type="button" onclick="alert('¡¡Aviso!!')"  value="Botón"><br>
</form>

label

Es una etiqueta que los lectores de pantallas lo leen, además ayuda en los botones radio ya que si haces click sobre la etiqueta, el boton radio se activará.

El atributo for es la relación con el input que tiene mediante la id de este.
